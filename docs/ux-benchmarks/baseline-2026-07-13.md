# Level CRE UX Baseline - 2026-07-13

## Scope

This baseline covers five deterministic broker journeys on desktop and mobile plus one deployed Google Maps smoke test. It is a mechanical workflow benchmark, not a score for the entire product or a replacement for Patrick's judgment.

## Deterministic Journeys

Result before deployment: 5/5 passed.

| Journey | Surface | Result | Console errors | Critical/serious axe findings |
| --- | --- | --- | ---: | ---: |
| Morning revenue check | Desktop | Passed | 0 | 0/0 |
| Create a listing pursuit | Desktop | Passed | 0 | 0/0 |
| Find captured deal activity | Desktop | Passed | 0 | 0/0 |
| Read the production scorecard | Desktop | Passed | 0 | 0/0 |
| Mobile follow-up check | Mobile, 390 x 844 | Passed | 0 | 0/0 |

The first run surfaced an unnamed toast close button, an unnamed Activity status filter, a restrictive viewport setting, and repeated low-contrast labels. Those issues were corrected before recording this baseline.

## Real Map

The deployed map completed Edmonton Places search, result selection, and aerial-mode switching. The run also exposed a Content Security Policy block on `https://places.googleapis.com`, which forced autocomplete onto the legacy compatibility path. The production policy was updated so the modern Places endpoint can be tested directly after deployment.
